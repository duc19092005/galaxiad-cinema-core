## 📌 Обзор проекта
Бэкенд-система **Cinema Booking Backend** построена на стеке **ASP.NET Core 8** и **SQL Server**, предоставляя полный набор API для управления современными кинотеатрами. Система обеспечивает высокий уровень взаимодействия с пользователями благодаря **блокировке мест в реальном времени**, интеграции защищенных электронных платежей, автоматическому контролю расписаний и фоновому планированию задач.

Проект разработан с использованием **Pragmatic Clean Architecture** (прагматичной чистой архитектуры), обеспечивая строгую модульность, разделение ответственности, высокую тестируемость и долгосрочную поддерживаемость кода.

## 🎯 Бизнес-ценность и ключевые возможности

Платформа переводит сложные бизнес-правила киноиндустрии в надежные автоматизированные процессы бэкенда:

### 1. Удобное бронирование и оплата в реальном времени
*   **Мульти-авторизация:** Поддерживает стандартную защищенную авторизацию через JWT (AccessToken и RefreshToken) и быструю интеграцию с **Google OAuth 2.0**.
*   **Интуитивный каталог сеансов:** API для поиска фильмов «В прокате» (Now-Showing) и «Скоро на экранах» (Coming-Soon) с возможностью гибкой фильтрации по кинотеатрам, датам и времени.
*   **Блокировка мест в реальном времени (SignalR):** В момент выбора места и перехода к оплате система **временно блокирует место на 10 минут** с помощью SignalR. Статус меняется в реальном времени на экранах всех остальных пользователей, исключая проблему **двойного бронирования** (Double Booking). Если оплата не завершается в течение 10 минут, фоновая задача Hangfire автоматически освобождает место.
*   **Электронные платежи:** Полная интеграция с платежным шлюзом **VNPay**, гарантирующая безопасные транзакции и автоматическое обновление статуса заказа через защищенный вебхук (callback).

### 2. Предотвращение конфликтов в расписании (алгоритм Breakdown Time)
*   **Интервал подготовки зала:** Бэкенд содержит автоматический валидатор расписания. Между любыми двумя сеансами в одном и том же зале должен быть обязательный перерыв не менее **15 минут (Breakdown Time)** для уборки и подготовки. Алгоритм на 100% исключает случайные накладки и пересечения фильмов по времени.

### 3. Динамический расчет стоимости билетов (Dynamic Pricing Engine)
Вместо фиксированных и жестко прописанных тарифов система использует интеллектуальный **механизм динамического ценообразования (Pricing Engine)**. Он оптимизирует доходность кинотеатров и повышает лояльность клиентов. Расчет стоимости билета разделен на два независимых этапа:

1. **Этап 1: Наценка за формат и зал (Surcharge Engine)**
   Позволяет максимизировать прибыль с каждого места на премиальных сеансах или в улучшенных залах (например: IMAX, 3D, VIP-залы, центральные кинотеатры).
   $$\text{Цена после наценки} = \text{Базовая цена} \times \left(1 + \frac{\text{Процент наценки}}{100}\right)$$
   *Наценки гибко настраиваются в зависимости от комбинации: `(Кинотеатр, Формат фильма 2D/3D/IMAX)`.*

2. **Этап 2: Скидка по сегменту лояльности (User Segment Discount)**
   Стимулирует повторные продажи и увеличивает коэффициент удержания клиентов (Customer Retention Rate), автоматически применяя скидку в зависимости от уровня лояльности пользователя при оплате.
   $$\text{Итоговая цена} = \text{Цена после наценки} \times \left(1 - \frac{\text{Процент скидки}}{100}\right)$$
   *Процент скидки определяется автоматически на основе сегмента пользователя: Standard (5%), Student (10%) или VIP (15%).*

### 4. Интеграция POS, управления отделами и учета рабочего времени (POS, Cashier Department & Attendance Flow)
*   **Синхронизация отделов касс (Cashier Department):** Поддерживает создание независимых кассовых отделов для каждого филиала. Автоматически создает общие учетные записи касс (роль `Cashier`) и гарантирует их блокировку (`AccountStatus = Banned`) и смену статуса персонала (`WorkingStatus = false`) через транзакцию БД при удалении или деактивации отдела.
*   **Биометрический учет времени (Face Match):** Сотрудники отмечают начало (Clock-In) и окончание (Clock-Out) смены непосредственно на POS-терминале с помощью распознавания лиц (сравнение 128-мерных векторов), без необходимости ввода личных учетных данных на самом терминале.
*   **Привязка кассира и оформление VIP-продаж (POS Checkout):** Четкое разделение продавца (StaffId - автоматически определяется по active смене, если не передан с фронтенда) и покупателя (UserId - определяется по электронной почте для начисления бонусов и применения скидок VIP 15% / Student 10%).

### 5. Фильтрация данных по кинотеатрам (Cinema-based Query Filtering)
*   **Фильтрация в каталоге фильмов:** Позволяет фильтровать список фильмов (`GET /api/movieManager/movies?cinemaId=...`), отображая только те фильмы, которые назначены для показа в конкретном выбранном кинотеатре.
*   **Фильтрация статистики Dashboard:** Администраторы могут фильтровать все данные на панели управления (`GET /api/v1/admin/dashboard/management?cinemaId=...`), чтобы отслеживать выручку, проданные билеты, графики по дням/часам, популярные фильмы, недавние транзакции и логи аудита отдельно по каждому кинотеатру.

## 👥 Модули и система прав доступа (Роли и Сегменты)

Контроль доступа строго разграничен в соответствии с бизнес-процессами кинотеатра на 6 системных ролей:

| Роль системы | Зона ответственности и возможности |
| :--- | :--- |
| **Customer** *(Клиент)* | Просмотр афиши, выбор мест, оплата билетов через VNPay, просмотр истории заказов и накопление бонусов. |
| **Admin** *(Администратор)* | Полное управление системой, блокировка учетных записей, настройка параметров и просмотр логов безопасности (Audit Logs). |
| **MovieManager** *(Контент-менеджер)* | Управление каталогом фильмов, форматами, загрузка постеров (интеграция с Cloudinary) и метаданных. |
| **TheaterManager** *(Менеджер расписания)* | Планирование сеансов для назначенных кинотеатров. Система автоматически применяет алгоритм защиты от накладок. |
| **FacilitiesManager** *(Управляющий залами)* | Управление залами, планировкой кресел (Standard, VIP, Couple) и техническим оборудованием. |
| **Cashier** *(Кассир)* | Продажа билетов и быстрое обслуживание клиентов на физической кассе кинотеатра. |

> 💡 **Особенность архитектуры:** Статусы `VIP` и `Student` **не являются системными ролями**. Это **Сегменты пользователей (User Segments)**. Такое разделение отделяет **права доступа в систему** (Роли) от **программ лояльности и ценообразования** (Сегменты), что упрощает добавление новых категорий (например, Senior, Child) без изменения кода авторизации.

### Сегменты пользователей и лояльность:
*   **Standard Member:** Скидка 5% на билеты, множитель бонусов 1.0x (назначается автоматически при регистрации).
*   **VIP Member:** Скидка 15% на билеты, множитель бонусов 2.0x (назначается администратором за объем покупок).
*   **Student:** Скидка 10% на билеты, множитель бонусов 1.5x.

## 🛠️ Технологический стек и архитектура

Качество разработки ориентировано на высокую производительность, безопасность и масштабируемость:

### 1. Стек технологий
*   **Язык и фреймворк:** C# / .NET 8, ASP.NET Core Web API
*   **База данных и ORM:** MS SQL Server, Entity Framework Core 8
*   **Обновления в реальном времени:** SignalR Hubs
*   **Планировщик фоновых задач:** Hangfire (автоматическое снятие брони с неоплаченных мест)
*   **Сторонние интеграции:**
    *   **VNPay:** Платежный шлюз.
    *   **Cloudinary:** Облачное хранилище медиафайлов и постеров.
    *   **Google OAuth 2.0:** Удобная авторизация пользователей.
*   **Документирование API:** Swagger OpenAPI (разделенный по модулям для удобного тестирования).

### 2. Clean Architecture (4-слойная структура)
Бэкенд состоит из 4 чистых слоев:
*   **`Cinema.Domain`**: Ядро чистого бизнеса, содержащее сущности (Entities), перечисления (Enums), пользовательские исключения (Exceptions), интерфейсы репозиториев/UnitOfWork, константы и утилиты. Не имеет внешних зависимостей.
*   **`Cinema.Application`**: Логика приложения, содержащая варианты использования (Use Cases), DTO, интерфейсы приложения и валидаторы.
*   **`Cinema.Infrastructure`**: Контекст базы данных, миграции, начальные данные (Seed Data), реализации репозиториев и интеграционные адаптеры (VNPay, Cloudinary, Redis, DeepSeek AI).
*   **`Cinema.Api`**: Слой представления API, содержащий контроллеры, промежуточное ПО (middlewares), конфигурации DI, хабы SignalR и Program.cs.

### 3. Инженерные преимущества
*   **Паттерны Unit of Work и Repository:** Гарантируют, что все изменения в рамках сессии бронирования (создание заказа, смена статуса мест, списание бонусов) проходят в рамках одной единой транзакции базы данных.
*   **Изоляция СУБД:** Бизнес-логика зависит только от интерфейса `IUnitOfWork`, что делает замену базы данных в будущем простой задачей.
*   **Dockerized:** Готовые файлы Dockerfile и Docker Compose для быстрого запуска системы в один клик в любой среде.
