# Стратегия кэширования Redis

В этом документе объясняется стратегия кэширования Redis, используемая в системе.

## Паттерн Cache-Aside

Система использует модель **Cache-Aside**:
1. При запросе данных система сначала проверяет Redis
2. При попадании в кэш данные возвращаются из Redis
3. При промахе кэша данные запрашиваются из базы данных и сохраняются в Redis
4. Данные кэшируются с временем жизни (TTL)

## Структура ключей

- `seat:temp_hold:{ScheduleId}:{SeatId}` — Временное удержание мест при бронировании
- `session:{UserId}` — Сессия входа пользователя
- Единый формат ключей для каждого типа данных

## Активное удаление кэша

При изменении данных (бронирование, обновление расписания и т.д.) соответствующий кэш немедленно удаляется для обеспечения актуальности данных.
