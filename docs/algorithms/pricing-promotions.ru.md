# Алгоритм динамического ценообразования и акций

В этом документе объясняется алгоритм автоматического расчета цены билетов.

## Основной принцип

Цена билета пересчитывается на бэкенде каждый раз, когда страница бронирования запрашивает цену, и снова при создании заказа. Фронтенд только отображает результат.

## Модель данных

- **PricingPromotionEntity**: Хранит информацию о кампании акции (название, URL-ярлык, заголовок, описание, изображение, срок действия)
- **PricingPromotionRuleEntity**: Хранит правила расчета (область применения, тип акции, значение, время/дата, дни недели, приоритет)
- **HolidayCalendarEntity**: Хранит праздничные дни Вьетнама

## Типы акций

1. **FixedTicketPrice**: Устанавливает фиксированную цену билета (например, 45 000 VND)
2. **PercentDiscount**: Вычитает процент из текущей цены (например, скидка 10%)
3. **FixedDiscount**: Вычитает фиксированную сумму (например, скидка 20 000 VND)
4. **Surcharge**: Добавляет процент к текущей цене (например, +10%)

## Процесс расчета

1. Начните с базовой цены формата фильма
2. Примените наценки по формату/кинотеатру/сегменту клиента
3. Найдите все подходящие активные правила акций
4. Если есть правило **FixedTicketPrice**, используйте эту цену (наивысший приоритет побеждает)
5. Если нет, примените правила скидок/наценок в порядке приоритета
6. Итоговая цена не может быть ниже нуля

## Битовая маска дней недели

Каждому дню недели присвоено битовое значение:
- Понедельник: 1, Вторник: 2, Среда: 4, Четверг: 8, Пятница: 16, Суббота: 32, Воскресенье: 64
- Хранится как целое число, сопоставление через битовое И

## Снимок бронирования

При бронировании бэкенд делает снимок примененных цен и акций для исторического хранения, чтобы старые цены билетов не менялись даже при последующем редактировании или удалении акции администратором.
